Users

The User object represents users within the system. Users tie together other Resources in the system, including Transfers, Accounts, and Likes. 

                  
   GET /users
  POST /users
   GET /users/:id
   PUT /users/:id

id integer
Unique identifier for the object.

firstName string
The user's first name.

lastName string
The user's last name.

transfers list
The user's transactions, if any.

likes list
The user's likes, if any. 

                  
{
  "id": 1,
  "firstName": "Jenny",
  "lastName": "Smith",
  "accounts": [111],
  "transfers": [1],
  "likes": [1, 12]
}

Accounts

The Account object represents bank accounts within the system. These Accounts are owned by Users and used to create Transfers. 

                  
   GET /accounts
  POST /accounts
   GET /accounts/:id
   PUT /accounts/:id

id integer
Unique identifier for the object.

user integer
The ID of the User this object belongs to.

accountNumber string
The bank account number.

balance integer
The account's balance in cents. 

                  
{
  "id": 13,
  "user": 1,
  "accountNumber": "1404291114",
  "balance": 5000
}

Transfers

The Transfer object represents bank transfers within the system. Transfers are between an origin account, the bank account the funds will be transfered from, and a target account, the bank account funds will be transfered into. The status of a Transfer will initially be 'initiated' and updated to 'complete' or 'failed' depending on the success of the transfer. 

                  
   GET /transfers
  POST /transfers
   GET /transfers/:id
   PUT /transfers/:id

id integer
Unique identifier for the object.

status string
The transfer's status.

originAccount integer
The ID of the origin Account for the object.

targetAccount integer
The ID of the target Account for the object.

amount integer
The transfer's amount in cents.

description string
The transfer's description, as provided by the user.

initiatedAt string
Time at which the transfer was initiated, ISO 8601 formatted.

completedAt string
Time at which the transfer was completed, ISO 8601 formatted.

failedAt string
Time at which the transfer failed, ISO 8601 formatted. 

                  
{
  "id": 1,
  "status": "complete",
  "originAccount": 111,
  "targetAccount": 212,
  "amount": 10000,
  "description": "An example of a completed transfer",
  "initiatedAt": "2019-05-20T12:53:38-04:00",
  "completedAt": "2019-05-22T12:00:00-04:00",
  "failedAt": null
}

Likes

The Like object represents likes within the system. Likes are created by users and associated with transfers. 

                  
   GET /likes
  POST /likes
   GET /likes/:id
   PUT /likes/:id

id integer
Unique identifier for the object.

user integer
The ID of the User that created the object.

transfer integer
The ID of the Transfer associated with the object. 

                  
{
  "id": 134,
  "user": 1,
  "transfer": 15
}
